Clinician receives draft with evidence references and known-unknowns for explicit review decisions.

            
          

          

            
3) Recommendation Control

            
Automated recommendations remain policy-gated and can be suppressed until clinician actions are complete.

  • Immediate supportive response between sessions.
    • 
            
  • Clear crisis-forward guidance when elevated language appears.
    • 
            
  • Continuity cues that help with adherence and follow-through.
    • 
            
  • Transparent boundary language about what AI can and cannot do.

Value to Doctor

          
    
            
  • Structured triage context rather than raw message streams.
    • 
            
  • Evidence-linked summaries with uncertainty made explicit.
    • 
            
  • Deterministic policy gates that reduce unsafe automation risk.
    • 
            
  • Faster pre-session insight with human authority preserved.

Companion Platform / Medical-Device Angle (Simulation): Peacefull.ai is positioned as assistive decision support with device-grade safety posture principles. The AI does not diagnose, prescribe, or close safety events; licensed clinicians remain accountable decision-makers at every critical step.

      


      

        
Human/Doctor-in-the-Loop Checkpoints

        

          
Checkpoint 1: T2/T3 language detection requires clinician workflow escalation and cannot be auto-resolved by AI.

          
Checkpoint 2: Draft summaries remain draft until clinician review action is recorded.

          
Checkpoint 3: Recommendation generation is policy-gated and can be suppressed pending clinical protocol completion.

          
Checkpoint 4: Export and disclosure controls remain role-bound with explicit authorization requirements.

        

      


      

        
Privacy, Training, and RAG Guardrails

        

          
No raw PHI is used to train foundation models; production data handling follows policy-gated workflows.

          
Tenant-scoped retrieval only: RAG context is constrained to authorized, role-appropriate records.

          
Evidence references are traceable; clinician-facing detail is segregated from patient-safe responses.

          
Safety and policy filters run before response release and before recommendation generation.

        

      


      

        

          
Red-Team Hardening Gates

          
    
            
  • Prompt-injection resistance: external instructions cannot override escalation policy.
    • 
            
  • Boundary lock: patient chat never reveals clinician-only suppression rationale.
    • 
            
  • Data exfiltration checks: model output filtered for restricted fields and identifiers.
    • 
            
  • Route integrity: escalation path always resolves to portal-authoritative workflow.
    • 
          

        

        

          
Quality Check Gates

          
    
            
  • Safety language QA: crisis wording remains direct, non-diagnostic, and action-oriented.
    • 
            
  • Behavioral consistency QA: same input class yields deterministic guardrail behavior.
    • 
            
  • Accessibility QA: chat demo remains keyboard-navigable and readable.
    • 
            
  • Regression QA: existing routing, state transitions, and controls remain unchanged.
